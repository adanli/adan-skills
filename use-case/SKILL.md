# 用例图生成功能

## 什么时候使用

当用户提到"请结合用例图，帮我生成功能清单"、"帮我生成功能清单"、"帮我生成功能文档"时，必须使用这个技能

## 前提条件

读取该技能所在的目录，记为USE_CASE_ROOT_PATH

## 工作流程

1. 分析用户提供的用例图，识别其中的参与者、系统、功能，并形成参与者-功能、系统-功能、参与者-系统-功能的关系，三者包含其一即可，需检查，用例图中所有的功能是否都已形成了关系
2. 读取USE_CASE_ROOT_PATH/assets/template.md文件，这个文件中记录了功能清单文档的模版
3. 根据功能清单文档的模版，将用例图中解析的关系，维护成一份功能清单

## 核心原则

1. 检查用例图，确保在功能清单中，不要遗漏任何的功能
2. 对于功能-功能的场景，解析到叶子结点，再回溯形成这条功能关系链
